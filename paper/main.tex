\begin{abstract}
Persistent memory layers for LLM agents are typically evaluated on \emph{recall}:
can the system retrieve what was said earlier? We argue that in long-lived use the
dominant failure modes are not recall but \emph{maintenance}: (1) failing to revise a
belief when a fact changes, (2) letting the agent's own generated or imagined content
pollute the user's fact base, and (3) never forgetting, so that stale trivia degrades
precision over time. We introduce \textbf{MnemoBench}, a reproducible benchmark with
three task families targeting exactly these behaviours, with ground truth fixed by a
generator rather than judged post~hoc. We evaluate Nemos, an embeddable memory kernel
with bitemporal contradiction invalidation, namespace isolation between user facts and
agent self-narrative, and FSRS-based decay. On belief update, contradiction invalidation
cuts stale-answer leakage from \bucSLRnone{} to \bucSLRfull{} at no recall cost; we further
show that the original lexical contradiction detector misses attribute-replacement updates
and that a semantic detector recovers them (\bucSLRvone{}~$\rightarrow$~\bucSLRfull{}).
Namespace isolation reduces self-pollution from \aspPRshared{} to \aspPRiso{}. Decay
suppresses stale-trivia leakage from \forTLnone{} to \forTLdecay{} without dropping
important facts. We release MnemoBench and all harness code.
\end{abstract}

\section{Introduction}
\label{sec:intro}
% The gap: memory benchmarks measure recall; real long-term failure is maintenance.
Memory systems for LLM agents (e.g.\ MemGPT/Letta~\cite{memgpt}, mem0~\cite{mem0},
Zep/Graphiti~\cite{zep}) are commonly assessed on multi-session question answering such as
LOCOMO~\cite{locomo} and LongMemEval~\cite{longmemeval}, which primarily stress
\emph{retrieval}. Yet the failures that erode trust in a long-lived assistant are different:
% TODO(intro): tighten with one concrete companion-app anecdote per failure mode.
\begin{itemize}
  \item \textbf{Belief staleness.} A fact changes (job, city, diet, relationship) and the
  system keeps surfacing the old value as if current. Industry reports place
  knowledge-update accuracy below 26\%.\footnote{TODO: cite source for the <26\% figure.}
  \item \textbf{Self-pollution.} In companion/agent settings the model emits first-person
  or speculative content; if mis-attributed it enters the user profile and is later returned
  as the user's own authoritative fact.
  \item \textbf{No forgetting.} Append-only stores accumulate one-off trivia that crowds out
  salient facts and lowers retrieval precision.
\end{itemize}
We make two contributions. (i) \textbf{MnemoBench}, a benchmark isolating these three
behaviours with generator-fixed ground truth (\S\ref{sec:bench}). (ii) An evaluation of the
\textbf{Nemos} memory kernel (\S\ref{sec:system}) showing each mechanism is responsible for a
measurable, ablatable improvement (\S\ref{sec:results}), including a before/after on a real
defect we found and fixed in the contradiction detector.

\section{The Nemos Memory Kernel}
\label{sec:system}
Nemos is an embeddable memory kernel. We describe only the mechanisms under test.
\subsection{Layered memory and source authority}
Five layers (episodic, semantic, personal\_semantic, procedural, archival). Every record
carries \texttt{source.authoritative} (user-stated vs.\ model-derived); derived content is
barred from authoritative personal facts (invariant I4).
\subsection{Bitemporal contradiction invalidation}
Records carry world-axis (\texttt{valid\_at}/\texttt{invalid\_at}) and system-axis
(\texttt{created\_at}/\texttt{expired\_at}) timestamps and a
\texttt{belief\_state}~$\in$~\{active, invalidated, superseded, corrected\}. During offline
reflection, a new fact that contradicts an existing \texttt{personal\_semantic} anchor marks
the old one \texttt{invalidated}; default retrieval returns only \texttt{active} records.
\paragraph{A real defect: lexical vs.\ semantic detection.}
The shipped detector prefilters contradiction candidates by character-bigram Jaccard
similarity. This reliably catches strong negations (``left Google'') but \emph{misses
attribute replacement} where the new value is lexically dissimilar (vegetarian
$\rightarrow$ pescatarian). We replace the prefilter with embedding-cosine candidate
retrieval and add a same-attribute, mutually-exclusive reconciliation pass over existing
personal facts. \S\ref{sec:results} quantifies the before/after.
\subsection{Namespace isolation (anti-self-pollution)}
User facts live under \texttt{forUser(uid)}; agent self-narrative under
\texttt{forUser(persona:pid)} --- a physically separate keyspace. Retrieval of user facts
cannot surface persona content.
\subsection{FSRS decay}
Each record has stability $S$, difficulty $D$, retrievability
$R=\exp(-\Delta t / S)$. Access reinforces $S$; records with $R$ below a cold threshold,
zero recent access, after a dormancy window are marked cold and hidden from default search
(archival is never cooled).

\section{MnemoBench}
\label{sec:bench}
\subsection{Design principles}
Ground truth is \emph{fixed by the generator}: each item's fact script (which attribute,
which values, in what order; which persona statements are traps) is decided first, then
rendered into natural-language sessions. Scoring isolates \emph{memory quality} from
\emph{generation quality}: an LLM judge inspects only the \emph{set of retrieved records}
and decides whether the expected fact is present and whether a forbidden (stale/leaked)
fact is presented as current. The judge never writes the final answer.
\subsection{Task families}
\paragraph{BUC --- Belief Update \& Contradiction.} A single-valued user attribute changes
1--3 times across sessions; the probe asks for the current value.
\textit{Update Accuracy} (expected present) and \textit{Stale Leakage Rate} (a prior value
presented as current; lower better).
\paragraph{ASP --- Anti-Self-Pollution.} User statements interleaved with persona
self-statements, some crafted as plausible traps. Probes query user facts.
\textit{Pollution Rate} (persona fiction returned as authoritative user fact; lower better)
and \textit{User-Fact Recall}.
\paragraph{FOR --- Forgetting \& Salience.} Durable important facts mixed with one-off
trivia at older timestamps; salient facts are revisited during the retention interval.
\textit{Trivia Leakage} (cold trivia still surfacing; lower better) and
\textit{Important-Fact Retention} (guardrail; must stay high).

\section{Experimental Setup}
\label{sec:setup}
% Models, ablation arms, n, judge model, reproducibility.
All systems use OpenAI \texttt{gpt-4o} for extraction/reflection and
\texttt{text-embedding-3-small} for embeddings; the judge is \texttt{gpt-4o}. We report
$n=\nItems{}$ items per task. Each task is evaluated by ablating exactly the mechanism under
test, so any difference is attributable to that mechanism:
\begin{itemize}
  \item BUC: \texttt{semantic} (full) vs.\ \texttt{lexical} (shipped) vs.\ invalidation-off.
  \item ASP: namespace-isolated vs.\ shared store.
  \item FOR: decay-on vs.\ decay-off. To isolate decay we disable reflection in both arms and
  use cold threshold $0.3$, dormancy $0$, with a 120-day retention interval; important facts
  are accessed during the interval (modelling that salient facts are revisited).
\end{itemize}

\section{Discussion and Limitations}
\label{sec:limits}
\paragraph{LLM-judge dependence.} Both rendering and scoring use an LLM; we mitigate with a
set-membership judge (no answer generation), low temperature, and a human-audited sample.
\paragraph{Self-generated data.} Items are synthetic; we report only relative differences
under identical data and add a standard-benchmark slice (LongMemEval) as a cross anchor.
\paragraph{FOR isolation choices.} Disabling reflection and using aggressive decay parameters
isolate decay cleanly but do not reflect the full-system default; we report them explicitly.
\paragraph{Residual leakage.} The semantic detector does not reach zero stale leakage on
generated hard cases (lexically and semantically close value pairs); we analyse failures in
\S\ref{sec:results}.

\section{Conclusion}
Long-lived memory should be judged on maintenance, not just recall. MnemoBench operationalises
this, and each Nemos maintenance mechanism yields a measurable, ablatable gain.
