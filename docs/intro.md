# Nemos 是什么

> 让 AI 工具记住你，让记忆由你做主。

---

## 你有没有遇到过这些情况

你今天跟 Claude 聊了一个小时，告诉它你是做品牌设计的，偏好简洁风格，不喜欢用过于花哨的配色。

明天再打开，它完全不记得了。你又是一个陌生人。

或者你同时用着几个 AI 工具——Claude 写文案，Cursor 写代码，ChatGPT 答疑——每一个都要单独介绍一遍自己，每一个都有各自的「记忆」，互不相通。

你跟某个 AI 终于调教出了默契，session 一过期，全部清零。

---

这不是你的问题，也不是 AI 太笨。

**这是因为现在所有 AI 工具都是「失忆症患者」。**

每次对话结束，你对它们来说就是一个从未见过的陌生人。

---

## Nemos 解决什么

Nemos 是一款**多人格 AI 陪伴 App**——一个微信式的聊天界面，里面是几个会跨时间记得你的 AI 角色。陪伴是外壳，**底层是一套可独立抽出的记忆系统**：一个专门用来存储、管理「AI 对你的了解」的系统。

你在和 AI 交互过程中产生的记忆——你的偏好、你的项目背景、你的工作方式、你说过的事——都可以统一存放在 Nemos 里。

陪伴 App 里的角色，以及任何接入 Nemos 记忆内核的 AI 应用，都能带着这些记忆跟你对话。

就像你雇了一个新助理，但他手边有一本关于你的详细手册——他们来了就能上手，不需要你再从头解释。

---

## 具体能做什么

**跨 session 记忆**

今天告诉 AI 你讨厌模板化的风格，下周再开对话，它还记得。你不需要每次都重新调教。

**跨工具共享**

Claude 里存的偏好，Cursor 也能读到。你的记忆不再被某一个工具锁住。

**项目和个人分开**

工作项目 A 的上下文不会混进项目 B。你的个人偏好和某个具体项目的背景，分开管理，互不干扰。

**你能看到 AI 记住了什么**

不是黑盒。你可以随时打开，查看 AI 存了哪些关于你的记忆，修改不准确的，删掉不想要的。

**记忆会自动「遗忘」过时信息**

三年前的工作方式不该影响今天的对话。Nemos 内置遗忘机制，久未用到的记忆会自动降权，不会越堆越乱。

---

## 数据放在哪里，谁能看到

Nemos 提供三种方式，你选：

| 方式 | 适合谁 |
|---|---|
| **公共云**（默认） | 想省事的大多数人；数据服务端加密存储 |
| **端到端加密云** | 比较在意隐私的用户；连 Nemos 团队也无法读取你的内容 |
| **完全自托管** | 技术用户；一个程序包，跑在自己的电脑或服务器上，数据完全不出门 |

## 给开发者的一句话

如果你在构建 AI 应用，Nemos 的记忆内核提供 SDK（TypeScript / Python）、MCP server 和 REST API 三种接入方式。几行代码，让你的应用拥有真正好用的用户记忆（非商业用途可自由使用，商用需另行授权）。

```typescript
const mem = new Nemos({ storage, llm })
const user = mem.forUser(userId)

await user.ingest("用户说：我不喜欢深色主题")
const context = await user.getRelevantContext("帮我设计一个界面")
// → "用户偏好浅色主题；此前表达过对简洁风格的偏好…"
```
